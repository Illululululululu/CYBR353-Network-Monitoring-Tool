Network Monitoring Tool

A packet-level network intrusion detection tool built with Scapy. Analyses PCAP files or live traffic for attack patterns, logs results with Fernet encryption, and generates human-readable reports.

File Structure

network-monitoring-tool/
│
├── compat.py             # Must be imported first — patches Scapy's IPv6 crash in sandboxed Linux
├── main.py               # CLI entry point (--pcap / --live / --decrypt)
├── capture.py            # PCAP file reader + live packet sniffer
├── analyzer.py           # Parses raw Scapy packets into structured dicts
├── detector.py           # Two-stage detection engine (regex + IP behaviour)
├── rules.py              # All detection patterns, suspicious ports, thresholds
├── secure_logger.py      # Fernet-encrypted append-only log writer/reader
├── report.py             # Generates human-readable text report from encrypted log

├── tests.py              # 35 unit + integration tests (no external test framework needed)
├── test_generate.py      # Generates samples/test.pcap with synthetic attack traffic

├── samples/

├── logs/

    └── report.txt        # Human-readable analysis report (created at runtime)

Dependencies

Package Version Purpose
scapy ≥ 2.5.0 Packet capture, parsing, and PCAP I/O
cryptography ≥ 41.0.0 Fernet symmetric encryption for log files

Both are listed in requirements.txt. Install with:

pip install -r requirements.txt

Note: Live capture (--live) requires root/administrator privileges because it opens a raw socket.

Module Overview

compat.py

Must be the first import in any file that uses Scapy. In restricted Linux environments (containers, CI sandboxes), Scapy’s rtnetlink module crashes on import because kernel netlink responses omit the scope key for IPv6 addresses. This shim pre-registers a safe stub for scapy.arch.linux.rtnetlink before Scapy initialises, exposing the four symbols the arch layer requires (read_routes, read_routes6, in6_getifaddr, _get_if_list). On a normal host machine the shim is a no-op.

rules.py

Central configuration file — edit this to tune the tool without touching logic: - SUSPICIOUS_PATTERNS — compiled regex dict for SQL Injection, Command Injection, Path Traversal, PHP/Web Shell, XSS, Suspicious Login - SUSPICIOUS_PORTS — set of port numbers that trigger a flag on their own (e.g. 23 Telnet, 445 SMB, 4444 Metasploit) - LARGE_PAYLOAD_THRESHOLD — packet size in bytes above which a “Large Payload” flag is raised (default: 10 000) - MAX_PACKETS, MAX_PORTS, MAX_DEST_IPS, MAX_ATTEMPTS — IP-behaviour thresholds for high-volume, port-scan, lateral-movement, and brute-force detection - compute_severity(reasons) — maps reason count → none / low / medium / high

analyzer.py

Converts a raw Scapy Packet object into a plain Python dict with keys: timestamp, src_ip, dst_ip, protocol, src_port, dst_port, tcp_flags, payload (UTF-8, errors ignored), packet_size. Returns None for non-IP frames.

detector.py

Two-stage engine:

Stage 1 — detect_packet(record) (stateless, per-packet) Runs all regex patterns against the payload, checks destination port against SUSPICIOUS_PORTS, and flags oversized packets. Returns {status, severity, reasons}.

Stage 2 — IPTracker (stateful, per-session) Maintains per-source-IP counters across all packets in a capture. Call tracker.update(record) for every packet, then tracker.check_ip(src_ip) to get behavioural flags. Detects: - High traffic volume (packet_count > MAX_PACKETS) - Port scan (len(dest_ports) > MAX_PORTS) - Lateral movement (len(dest_ips) > MAX_DEST_IPS) - Brute-force / repeated attempts (max connection_attempts > MAX_ATTEMPTS)

merge_results(pkt_result, ip_flags) combines both stages into one final result dict.

capture.py

  • read_pcap(path, callback) — reads a .pcap file and calls callback(pkt) for every packet
  • live_sniff(callback, count, iface, bpf_filter) — wraps scapy.sniff() with sensible defaults; supports BPF filters and interface selection

secure_logger.py

  • SecureLogger(log_path, key_path) — context-manager-aware writer; serialises each record to JSON, encrypts with Fernet, and appends one Base64 token per line
  • decrypt_log(log_path, key_path) — generator that yields decrypted dicts; skips and warns on corrupt lines
  • load_or_create_key(key_path) — reads key.key or generates a fresh Fernet key and saves it

⚠️ Losing key.key makes all encrypted logs permanently unreadable. Back it up separately from the log files.

report.py

generate_report(log_path, report_path, key_path) — iterates the encrypted log, computes: - Packet totals (normal / suspicious) - Suspicious source IPs with deduplicated reason list (port-scan counts collapsed to highest seen) - Top 10 detection reasons by frequency - Up to 10 sample suspicious packet summaries

Writes the report to disk and prints it to stdout.

main.py

CLI entry point. Parses arguments and dispatches to the correct mode.

Usage

Analyse a PCAP file

python main.py --pcap samples/test.pcap

Live capture (root required)

sudo python main.py --live --count 200

Live capture with interface and BPF filter

sudo python main.py --live --count 500 --iface eth0 --filter "tcp port 80"

Generate a report from an encrypted log

python main.py --decrypt logs/traffic.enc --report reports/report.txt

Full example with custom paths

sudo python main.py --live --count 100 --log logs/run1.enc --key secrets/run1.key
python main.py --decrypt logs/run1.enc --report reports/run1.txt --key secrets/run1.key

All CLI flags

Flag Default Description
--pcap FILE — Analyse a PCAP capture file
--live — Capture from a live interface
--count N 100 Packets to capture in live mode
--iface IFACE Scapy default Network interface for live sniffing
--filter BPF — BPF filter string, e.g. tcp port 443
--decrypt FILE — Encrypted log to decrypt and report on
--report FILE reports/report.txt Report output path
--log FILE logs/traffic.enc Encrypted log output path
--key FILE key.key Fernet key file path
